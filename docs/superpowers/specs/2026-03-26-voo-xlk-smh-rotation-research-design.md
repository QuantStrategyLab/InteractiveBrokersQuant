# VOO XLK SMH Rotation Research Design

## 1. 背景

当前实盘默认版本已经把 `QQQ` 纳入轮动池，作为“稳健但不完全排除科技”的方案。

新的研究目标不是继续堆叠 `QQQ`，而是把美国宽基、科技、半导体拆开：

- 地区/大盘类：`EWY / EWT / INDA / FXI / EWJ / VGK / VOO`
- 行业类：`XLE / XLF / ITA / XLP / XLU / XLV / IHI / VNQ / KRE / XLK / SMH`

然后把这两类 ETF 放进一个总池里，继续按现有策略规则统一做 `Top 2` 轮动。

这样做的核心想法是：

1. 保留“全球 ETF 轮动”的框架
2. 允许美国大盘和科技行情进入策略
3. 避免 `QQQ` 同时承担“美国大盘 + 科技成长”的双重角色，降低解释上的混合

## 2. 目标

本轮只做研究，不直接改实盘默认策略。

需要回答的问题：

1. 用 `VOO + XLK + SMH` 拆分 `QQQ` 后，统一放在一个总池做 `Top 2`，长期收益和回撤是否优于当前默认 `QQQ` 版
2. 改善如果存在，主要来自：
   - `VOO` 这个美国宽基
   - `XLK` 这个科技行业
   - `SMH` 这个半导体行业
3. 这套拆分方案是否值得在后续替换当前默认实盘池

## 3. 非目标

这轮先不做这些事：

1. 不改实盘默认策略代码
2. 不改下单逻辑、动量公式、金丝雀规则、调仓频率
3. 不引入“两个池子各选一个”的新机制
4. 不在第一轮加入模拟历史
5. 不把全球科技或全球半导体 ETF 直接并入主研究结论

## 4. 研究方案

### 4.1 主对比

第一层输出直接回答“新总池值不值得换”。

需要比较三组主策略：

1. `legacy_non_tech`
   - 旧版非科技池
2. `current_default_qqq`
   - 当前默认 `QQQ` 版
3. `proposed_voo_xlk_smh`
   - 新总池
   - 候选池：
     `EWY / EWT / INDA / FXI / EWJ / VGK / VOO / XLE / XLF / ITA / XLP / XLU / XLV / IHI / VNQ / KRE / XLK / SMH`

### 4.2 拆解对比

第二层输出回答“收益变化主要来自谁”。

新增三组拆解策略：

1. `replace_qqq_with_voo`
   - 在当前默认 `QQQ` 版基础上，用 `VOO` 替换 `QQQ`
2. `voo_plus_xlk`
   - 在 `VOO` 基础上加入 `XLK`
3. `voo_plus_xlk_plus_smh`
   - 在 `VOO + XLK` 基础上再加入 `SMH`

这样可以逐步观察：

- 仅换成美国宽基有没有改善
- 仅加科技行业有没有额外提升
- 半导体是不是进一步拉高收益，还是只是增加波动

## 5. 回测规则

继续复用现有研究脚本里的规则，不新增新逻辑：

1. 13612W 动量
2. `SMA200` 趋势过滤
3. 持仓 `+2%` bonus
4. 季度调仓
5. 金丝雀应急
6. `Top 2` 等权
7. 不足两个标的时补 `BIL`

原因是这轮要比较的是“候选池结构”，不是“策略规则变化”。

## 6. 输出结果

继续沿用现有脚本输出，避免口径漂移。

### 6.1 全样本指标

1. Total Return
2. CAGR
3. Max Drawdown
4. Volatility
5. Sharpe
6. 与 `SPY` / `QQQ` 的相关性

### 6.2 关键阶段

先看这几个区间：

1. 全样本公共区间
2. `2022`
3. `2023+`

如果脚本里保留原有的 `Tech Bull 2009-2021` 标签，可以继续打印，但解释时要明确实际公共起点，以免口径误导。

### 6.3 年度收益

保留年度收益表，用来观察：

1. `2022` 这样的逆风年是不是明显更差
2. `2023+` 这种科技驱动阶段是不是明显更强
3. 提升是不是集中在少数年份

## 7. 数据与历史边界

### 7.1 第一轮不做模拟

这轮不为 `VOO`、`XLK`、`SMH` 做模拟历史。

原因：

1. 这三只 ETF 的真实历史已足够覆盖当前公共回测区间
2. 真实 ETF 数据的结论更容易解释
3. 先回答“真实可交易组合是否值得换”比延长样本更重要

### 7.2 第二轮再考虑模拟

如果后续继续研究“全球科技 / 全球半导体”候选，再单独引入代理或模拟规则。

到那时需要明确记录：

1. 哪个标的用了代理
2. 代理来源是什么
3. 从哪一天开始代理
4. 为什么选择这个代理

并且把“真实历史结果”和“含模拟历史结果”分开打印。

## 8. 技术实现

### 8.1 复用现有脚本

优先修改现有：

- `research/backtest_qqq_variants.py`

而不是新增第二套研究脚本。

原因：

1. 现有脚本已经包含策略规则、下载、对齐、汇总和打印逻辑
2. 这轮只是新增配置矩阵，不是新研究框架
3. 复用脚本更容易直接对比旧版、`QQQ` 版和新结构版

### 8.2 配置扩展

在现有脚本里新增配置集合：

1. `legacy_non_tech`
2. `current_default_qqq`
3. `proposed_voo_xlk_smh`
4. `replace_qqq_with_voo`
5. `voo_plus_xlk`
6. `voo_plus_xlk_plus_smh`

必要时把脚本内部的配置构建从“专门服务 QQQ 研究”收敛成“通用策略配置列表”。

但要保持改动尽量小，不做无关重构。

## 9. 风险

1. `VOO`、`XLK`、`SMH` 都是美股方向，虽然比 `QQQ` 可解释性更强，但整体组合对美股风格的依赖会更高
2. `SMH` 不是严格意义上的“全球半导体 ETF”，更像半导体研究代理
3. 如果结果提升主要集中在 `2023+`，要警惕样本偏向最近科技行情
4. 如果新方案显著提高和 `QQQ` 的相关性，虽然收益更高，但“稳健策略”的定位可能会变弱

## 10. 验收标准

完成后应该能明确回答：

1. 新总池是否优于当前默认 `QQQ` 版
2. 改善是否主要来自 `VOO`、`XLK`、`SMH` 中的某一个或某两个
3. 新总池是否值得进入下一轮实盘候选讨论
4. 后续是否值得继续研究全球科技或全球半导体 ETF 代理方案
